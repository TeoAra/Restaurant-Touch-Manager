Voglio costruire un software gestionale POS per ristorante/pizzeria/bar, professionale, scalabile e pensato per funzionare in locale su Windows, con accesso via browser da tablet e PC in rete Wi-Fi.

Obiettivo:
Creare un sistema composto da:
1. POS operativo per sala/cassa
2. Web-admin per configurazione e gestione
3. Backend API locale
4. Database
5. Architettura pronta per stampanti di reparto Sewoo e fiscale DTR
6. Possibilità futura di installer Windows e vendita del software

Stack richiesto:
- Realtime pronto per futuro uso: Socket.IO o WebSocket
- UI moderna, pulita, touch-first, colori chiari
- Monorepo con struttura chiara

Requisiti architetturali:
- Il backend deve esporre API REST
- Il POS deve collegarsi al backend locale via HTTP
- Il sistema deve essere locale-first
- Il backend deve essere pensato per girare su un PC Windows locale
- In futuro il backend dovrà poter parlare con:
  - stampanti Sewoo via TCP/IP ESC/POS
  - registratore fiscale DTR tramite adapter separato
- Separare la logica hardware dal resto del sistema
- Preparare moduli dedicati per stampa e fiscale, anche se inizialmente mock
- l'accesso al frontend/backend deve essere tramite codice operatore quindi admin piuttosto che cameriere 
Voglio che il progetto venga generato come base reale e ordinata, non come demo casuale.

========================
LOGICA POS
========================

Flusso richiesto:
1. carico tavoli
2. seleziono tavolo
3. apro sessione tavolo
4. seleziono coperti (da impostare prezzo in backend)
4. creo ordine
5. aggiungo prodotti
6. salvo righe ordine nel DB
7. invio comanda
8. le righe passano da draft a sent

Per ora il pulsante “Invia comanda” deve:
- cambiare stato righe ordine in sent
- restituire messaggio di conferma
- preparare il terreno per futura stampa su reparti
